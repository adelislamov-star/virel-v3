ЧТО НЕЛЬЗЯ ТРОГАТЬ ПРИ РЕДИЗАЙНЕ
🔴 АБСОЛЮТНЫЙ ЗАПРЕТ — сломает SEO мгновенно:
1. URL-структура — ни один роут не переименовывать:

/london/mayfair-escorts → остаётся как есть
/companions/[slug] → slug модели не менять
/categories/[slug] → slug категории не менять
/blog/[slug] → slug статьи не менять

2. <title> теги — не трогать без причины, формат: Keyword | Vaurel
3. <meta name="description"> на каждой странице
4. <link rel="canonical"> теги
5. <h1> — один на страницу, строго по тексту (менять только если есть причина SEO, не ради дизайна)
6. robots.txt — не добавлять новых Disallow без проверки
7. Schema.org JSON-LD — не удалять блоки <script type="application/ld+json">
8. next.config.js — не трогать блок redirects и trailingSlash: false

🟡 МЕНЯТЬ ОСТОРОЖНО — влияет на SEO косвенно:
9. <h2> заголовки на district-страницах — они оптимизированы, менять только если добавляешь контент
10. Внутренние ссылки — блок profile-seo-links на профилях моделей не удалять
11. alt у изображений — когда будешь добавлять фото моделей, обязательно заполнять: alt="Marsalina — London companion Vaurel"
12. Sitemap-роуты (/blog/sitemap.xml, /london/sitemap.xml и т.д.) — не переносить, не переименовывать

🟢 ТРОГАЙ СВОБОДНО — на SEO не влияет:

Всё CSS и Tailwind классы
Layout, padding, цвета, шрифты
Анимации и transitions
Порядок и визуальное расположение элементов
Иконки, изображения (с условием про alt)
Навигация и footer (структура ссылок, не URL)
Любые React компоненты — при условии что title/h1/canonical остаются


При работе с изображениями:

Не использовать <img> напрямую — только Next.js <Image> с alt
Если добавляешь hero-изображения или фото моделей — priority={true} на первый экран (влияет на LCP / Core Web Vitals)

При работе с компонентами:

Если выносишь H1 в отдельный компонент — убедись что он рендерится на сервере (не "use client" с useEffect), иначе Google может не увидеть
Не оборачивай <h1> в display: none или visibility: hidden ради дизайна — это воспринимается как клоакинг

При работе со страницами:

Если создаёшь новые страницы (например /rates, /join) — сразу добавляй их в sitemap.xml и прописывай metadata с title/description
