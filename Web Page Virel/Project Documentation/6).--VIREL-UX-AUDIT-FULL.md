# UX/UI Аудит: Virel — virel-v3.vercel.app
### Жёсткий, объективный разбор. Без поддакивания.

**Дата аудита:** Март 2026
**Метод:** Живой просмотр всех страниц (главная, /london-escorts, /catalog/marzena, /services, mobile 390px)
**Стек:** Next.js 14, TypeScript, Tailwind CSS, Prisma, PostgreSQL, Vercel

---

## 1. ОБЩАЯ ОЦЕНКА

Сайт технически собран — работает, грузится, деплоится. Но продуктовая логика и UX разваливаются. Главная перегружена (11 секций вместо 5–6), цена спрятана за 5 экранами скролла, фотографии разного качества стоят рядом, навигация путает клиента с моделью-соискательницей. Ощущение premium отсутствует — сайт выглядит как шаблон, а не как бренд.

Если коротко: клиент заходит, не понимает сколько стоит, не чувствует доверия, листает мёртвые блоки — и закрывает вкладку.

---

## 2. ВИЗУАЛЬНАЯ ИЕРАРХИЯ: ОТСУТСТВУЕТ

### Что видит пользователь при скролле главной:

1. Hero (чёрный фон + текст, без фото)
2. Три бейджа (100% Verified / 24/7 Available / 30min Confirmation)
3. Три USP-блока (Fully Verified / Absolute Discretion / Curated Selection)
4. «Available Now» — метка секции
5. «Featured Companions» — заголовок
6. 6 карточек моделей (3×2)
7. «View All Companions» — кнопка
8. «How to Book» — три шага (01/02/03)
9. «Escorts by District» — SEO-сетка 9 районов
10. «Frequently Asked» — 6 вопросов в аккордеоне
11. Footer — 4 колонки

**11 смысловых блоков.** Hermès на главной показывает 3–4. Каждый — с воздухом, паузой, одним действием. У тебя — информационный поток, где всё одинаково важно, а значит — ничего не важно. Глаз не знает куда смотреть, рука не знает куда кликать.

**Принцип luxury:** меньше элементов → больше ценности каждого. Каждый экран — одна мысль, одно действие.

## 3. HERO-СЕКЦИЯ: ПУСТАЯ И ПЕРЕГРУЖЕННАЯ ОДНОВРЕМЕННО

### Что там сейчас:

```
[декоративная grid-сетка на чёрном фоне]

● LONDON'S PREMIER COMPANION AGENCY     ← badge
  
Elite London                             ← H1
Escorts                                  ← H1, italic, золотой

VERIFIED · SOPHISTICATED · DISCREET      ← подзаголовок

[BROWSE COMPANIONS]  [PRIVATE ENQUIRY]   ← два CTA
```

### Проблемы:

**Нет фотографии.** Сайт продаёт внешность — визуальный продукт. Первый экран: чёрный фон с типографикой. Для интернет-магазина одежды это было бы как главная без фото товара. Hermès на первом экране показывает продукт. Aman Resorts показывает отель. Net-a-Porter показывает модель в одежде. Virel показывает шрифт Cormorant Garamond на чёрном фоне.

**Два конкурирующих заголовка.** «London's Premier Companion Agency» (badge) + «Elite London Escorts» (H1). Оба претендуют на внимание. Нужен один H1 и один подзаголовок. Не два заголовка.

**Три прилагательных вместо ценности.** «Verified · Sophisticated · Discreet» — свойства, не выгода. Клиенту всё равно что ты «sophisticated». Ему важно: быстро, надёжно, сколько стоит. Hermès не пишет «Quality · Exclusive · French». Они создают ощущение через дизайн.

**Два CTA на первом экране.** «Browse Companions» (золотая заливка) + «Private Enquiry» (outline). Пользователь, который только зашёл, ещё не готов к «Private Enquiry». Это CTA для тёплого клиента, а не для холодного трафика с Google. Два CTA = ни одного решения.

**Декоративная grid-сетка.** Тонкие линии на фоне hero — выглядит как шаблон из Figma Community. Не несёт смысла. Визуальный мусор.

**Нет цены.** Первый экран не даёт ни одного ценового ориентира. Клиент с бюджетом £200 и клиент с бюджетом £5,000 видят одно и то же. Ни один не квалифицирован.

## 4. КАРТОЧКИ МОДЕЛЕЙ: НЕТ КЛЮЧЕВОЙ ИНФОРМАЦИИ

### Что показывает карточка:

```
[ФОТО]
Name
XX yrs · Nationality
```

### Что отсутствует:

**Цена.** Клиент не может понять, может ли он себе позволить companion, не кликнув на профиль, не проскроллив 5 экранов до формы букинга. Это главный убийца конверсии. При 50+ моделях без цены на карточках сравнение невозможно.

**Статус доступности.** «Available Now» заявлен как метка секции, но не показан на конкретных карточках. Клиент не знает, кто свободен прямо сейчас.

**Район/локация.** Клиент в Mayfair хочет companion в Mayfair. Без этой информации на карточке он вынужден открывать каждый профиль.

**Услуги/теги.** Клиент ищет конкретный experience. Без тегов — слепой выбор.

### Качество фотографий — катастрофа:

Я увидел на карточках Featured Companions:

| Модель | Качество фото | Проблема |
|--------|---------------|----------|
| Marzena | Профессиональная студия | ✅ Ок |
| Anser | Селфи в зеркале, телефон в кадре, наушники | ❌ Не premium |
| ISMAT | Коридор с жёлтым искусственным светом | ❌ Не premium |
| Bamerta | Нормальная студия | ✅ Ок |
| Comely | Профессиональная студия | ✅ Ок |
| Inglote | Фото из бара/клуба, неоновый свет, розовая юбка | ❌ Не premium |

**3 из 6 карточек на главной** — фото непрофессионального качества. Это стоит рядом со студийными снимками. Эффект: сайт выглядит как доска объявлений, а не как curated selection. Hermès не покажет одну сумку в студии, а другую — в селфи из метро. Визуальная консистентность = доверие = конверсия.

## 5. НАВИГАЦИЯ: ПУТАЕТ КЛИЕНТА С СОИСКАТЕЛЬНИЦЕЙ

### Текущее меню:

```
Virel     COMPANIONS  SERVICES  FAQ  CONTACT  [JOIN US]
```

### Проблемы:

**«Join Us» — самый заметный элемент.** Единственный с рамкой (золотой border). Визуально он главный. Но это ссылка для моделей, не для клиентов. Клиент, который платит £500+/час, видит рекрутинговую кнопку как основной CTA сайта. Это сбивает позиционирование и уровень.

**«Services»** — непонятно что это. Клиент пришёл за companion, не за «услугой». В luxury-сегменте не продают services — продают experience. Страница /services показывает категории (Classic, Massage, Fetish & Fantasy, BDSM) в сетке карточек. Но эти карточки не привязаны к моделям — клиент видит услугу, но не знает кто её предлагает. Бесполезный промежуточный шаг.

**«FAQ» в основном меню** — признак того, что основной контент не отвечает на вопросы. В luxury FAQ прячут в footer.

**Нет «Book Now».** Основное действие сайта — букинг. Но в навигации нет прямого CTA на букинг. Есть «Join Us» (для моделей), но нет «Book» (для клиентов).

## 6. СТРАНИЦА ПРОФИЛЯ МОДЕЛИ: 5 ЭКРАНОВ ДО ЦЕНЫ

### Путь клиента на профиле Marzena:

```
Экран 1: Hero-фото на весь экран (красивое, профессиональное)
Экран 2: Имя + «29 yrs · Brazilian · London» + параметры (163cm, 34C, Dark brown) + «Arrange a Meeting» кнопка справа
Экран 3: Portfolio — горизонтальный скролл 9 фото, «Drag to Explore»
Экран 4: Bio текст (italic) + таблица параметров (Age, Height, Figure, Hair/Eyes, Nationality, Languages)
Экран 5: НАКОНЕЦ — форма букинга «Arrange a Meeting» с ценами
```

**Клиент узнаёт цену через 5 экранов скролла.** Это критический провал. Человек, который хочет забукировать за £350/час, вынужден сначала:
- Посмотреть fullscreen-фото
- Прочитать параметры
- Пролистать 9 фото в карусели
- Прочитать bio
- И только потом увидеть, что 1 час стоит £350

Каждый из этих экранов — точка потери. Клиент может уйти на любом этапе, так и не узнав цену.

### Форма букинга — проблемы:

**Location (Incall/Outcall):** Два варианта в карточках — ок, но цена меняется в зависимости от выбора, а клиент этого не видит сразу.

**Duration:** 7 кнопок-чипов в сетке (30min £300 / 45min £330 / 1hr £350 / 1.5hr £500 / 2hr £650 / Extra hour £300 / Overnight £2,600). Формат «£2 600» вместо «£2,600» — непрофессионально.

**Date & Time:** Текстовые поля «DD/MM/YYYY» и «HH:MM». Не datepicker, не dropdown. Клиент должен вручную вбивать дату. Это гарантия ошибок и фрустрации.

**Contact:** Name, Phone, Notes — нормально.

**Submit:** «SELECT A DURATION TO SEE YOUR SUMMARY» → «SUBMIT YOUR REQUEST». Цена не показана в кнопке — клиент не получает финального подтверждения суммы.

### Sticky bar:

Золотая полноширинная плашка «ARRANGE A MEETING WITH MARZENA» перекрывает нижнюю часть экрана постоянно, на всём протяжении скролла. Для premium это агрессивно. Hermès не ставит «BUY THIS BAG NOW» поверх контента. Это тактика масс-маркета.

## 7. КАТАЛОГ (/london-escorts): РАБОТАЕТ, НО БЕЗ ЦЕНЫ

### Что хорошо:
- Breadcrumb (HOME → COMPANIONS)
- Фильтры районов (горизонтальный скролл)
- Кнопка «Filter»
- Счётчик «9 of 9 companions»
- Карточки 3 в ряд

### Что плохо:
- **Нет цены на карточках** — невозможно сравнить
- **Нет фильтра по цене** — при 50+ моделях критично
- **Нет фильтра по доступности** (Available Now)
- **Нет фильтра по услугам**
- Фильтры районов в горизонтальном скролле — «Canary Wharf» обрезается, не видно полностью

## 8. SERVICES (/services): ОСТРОВ БЕЗ СВЯЗИ

### Что показывает:

Заголовок «Companion Services in London». 6 категорий-фильтров (Classic Services, Massage, Companionship, Fetish & Fantasy, BDSM & Domination, Entertainment). Сетка карточек с названиями услуг и описаниями. «Classic Services — 14 services».

### Проблема:

**Услуги не привязаны к моделям.** Клиент видит «Deep Throat» как карточку с описанием, но не знает, кто из моделей это предлагает. Чтобы узнать — нужно вернуться в каталог, открыть профиль каждой модели, прочитать. При 50+ моделях и 40+ услугах это нерабочая схема.

**Нет обратной связи.** На профиле модели нет тегов услуг. Двусторонняя связь model↔service отсутствует.

**SEO-потенциал не реализован.** Страница /services/massage могла бы показывать моделей, предлагающих массаж, и собирать органический трафик. Сейчас это мёртвая страница с описаниями.

## 9. MOBILE (390px): USP = 3 ЭКРАНА МЁРТВОГО СКРОЛЛА

### Что хорошо:
- Hero адаптируется нормально
- Карточки переключаются на 1 колонку
- Бургер-меню работает

### Что плохо:
- **Три USP-блока (Fully Verified / Discretion / Selection) превращаются в 3 полных экрана скролла.** На десктопе они в ряд — терпимо. На мобильном каждый блок = полный экран. Клиент скроллит 3 экрана текста, который говорит одно и то же, прежде чем увидит хоть одну модель.
- Sticky bar занимает значительную часть маленького экрана
- Форма букинга — текстовые поля на мобильном вместо нативных пикеров

## 10. ДОВЕРИЕ: НЕТ SOCIAL PROOF

На всём сайте:
- ❌ Ни одного отзыва
- ❌ Ни одного упоминания в прессе
- ❌ Ни «Trusted by X clients»
- ❌ Ни «Est. 2024»
- ❌ Ни одного знака доверия (кроме повторения слова «verified» 4 раза)

Клиент видит сайт без истории, без социального подтверждения. В premium-сегменте, где средний чек £500+, это критический пробел. Клиент должен чувствовать, что это устоявшийся, надёжный сервис, а не сайт, запущенный вчера.

## 11. ОЩУЩЕНИЕ PREMIUM: ОТСУТСТВУЕТ

### Что делает сайт НЕ premium:

1. **Разнобой фото.** Селфи из зеркала рядом со студийными — уровень classified ads, не luxury agency.

2. **Hero без фото.** Пустой чёрный экран с текстом — это шаблон, не бренд.

3. **11 секций на главной.** Premium = мало элементов, много воздуха. Тут наоборот — много элементов, мало воздуха.

4. **Золотой цвет везде.** Используется на: заголовках, CTA, badge, «Available Now», стрелках, sticky bar, ценах, иконках, hover-состояниях. Когда акцентный цвет на всём — он перестаёт быть акцентным. Это как если бы в ресторане все блюда были «specialty» — тогда ни одно не specialty.

5. **Italic везде.** Cormorant Garamond italic используется для: H1, H2, секционных меток, bio, подзаголовков. Когда всё курсивом — ничего не выделяется.

6. **CAPS LOCK в CTA.** «BROWSE COMPANIONS», «PRIVATE ENQUIRY», «VIEW ALL COMPANIONS», «SUBMIT YOUR REQUEST» — кричащие кнопки. Premium не кричит. Premium говорит тихо, и к нему наклоняются.

7. **Агрессивный sticky bar.** Золотая полоса на весь экран — тактика e-commerce, не luxury.

8. **Дублирование USP.** «Verified» написано 4 раза на главной. Повторение обесценивает.

### Что делает Hermès (и что можно взять):

1. **Один продукт — один экран.** Не 6 карточек, а одно крупное фото.
2. **Воздух между элементами = ощущение дороговизны.** Пустое пространство стоит дороже контента.
3. **Фото — главный элемент, текст — поддержка.** У Virel наоборот: текст доминирует, фото вторичны.
4. **Тихий CTA.** Кнопки без caps lock, без золотого фона, без sticky bars.
5. **Консистентность.** Каждое фото одинакового качества и стиля. Без исключений.

## 12. ПУТЬ ПОЛЬЗОВАТЕЛЯ: ОТ ВХОДА ДО ОПЛАТЫ (ТЕКУЩИЙ)

```
1. Заходит на главную (3 сек)
   → Видит: чёрный экран + текст + два CTA
   → Не понимает цену, не видит фото
   → Может уйти (bounce rate ~40-50%)

2. Кликает «Browse Companions» (если не ушёл)
   → Попадает на /london-escorts
   → Видит 9 карточек БЕЗ ЦЕН
   → Должен кликнуть на карточку чтобы хоть что-то узнать

3. Кликает на модель (например Marzena)
   → Экран 1: fullscreen фото (красиво, но нет инфо)
   → Экран 2: имя, параметры, кнопка «Arrange a Meeting»
   → Экран 3: ещё 9 фото в карусели
   → Экран 4: bio + таблица параметров
   → Экран 5: НАКОНЕЦ — форма с ценами

4. Видит цену (£300-£2,600)
   → Выбирает Incall/Outcall
   → Выбирает Duration (кнопки-чипы)
   → Вбивает дату ВРУЧНУЮ (DD/MM/YYYY)
   → Вбивает время ВРУЧНУЮ (HH:MM)
   → Вводит имя, телефон
   → Жмёт «Submit Your Request»

5. Итого: ~5-7 минут от входа до submit
   → Каждый экран — точка потери
   → Цена видна только на шаге 4 из 5
```

## 13. ПУТЬ ПОЛЬЗОВАТЕЛЯ: ИДЕАЛЬНЫЙ СЦЕНАРИЙ

```
1. ВХОД (3 сек)
   → Видит: красивое hero-фото + «London's Finest Companions» + «From £300/hour»
   → Сразу понимает: что это, для кого, сколько стоит
   → Один CTA: [Explore Companions]
   → Кликает

2. КАТАЛОГ (15-30 сек)
   → 3 колонки карточек: фото + имя + «From £XXX/hr» + ● Available
   → Фильтры: район, цена, доступность
   → Сканирует цены, СРАВНИВАЕТ — не открывая профили
   → Кликает на карточку

3. ПРОФИЛЬ (30-60 сек)
   → Первый экран: фото 60% + справа: имя, «FROM £300/HOUR», [Book Now], ● Available
   → Цена видна СРАЗУ. Никакого скролл-марафона.
   → Скроллит если хочет: gallery, bio, rates table
   → Видит теги услуг: [GFE] [Dinner Date] [Massage]

4. RATES TABLE (10 сек)
   → Чистая таблица: Duration × Incall/Outcall
   → Видит: 1 hour Incall £350, Outcall £400
   → Решение по бюджету — мгновенно

5. ФОРМА (30 сек)
   → Date (календарь) → Time (dropdown) → Duration (select)
   → Name → Phone → Notes (optional)
   → Кнопка: [Request Booking — £350] ← цена в кнопке
   → Клик

6. ПОДТВЕРЖДЕНИЕ
   → «Thank you. Confirmation within 30 minutes.»
   → «While you wait — meet similar companions»

ОБЩЕЕ ВРЕМЯ: 2-3 минуты.
ТЕКУЩЕЕ ВРЕМЯ: 5-7 минут.
ТОЧКИ ПОТЕРИ: сокращены с ~8 до ~3.
```

## 17. РЕКОМЕНДАЦИИ ПО РАСПОЛОЖЕНИЮ ЦЕНЫ

| Где | Формат | Зачем |
|-----|--------|-------|
| Hero главной | «From £300 per hour» под H1 | Ценовой якорь с первого экрана, квалификация трафика |
| Карточка в каталоге | «From £300/hr» мелко под именем | Сравнение без открытия профиля |
| Hero профиля | «FROM £300/HOUR» крупно, рядом с CTA | Снять неопределённость на первом экране |
| Rates Table | Полная таблица Duration × Incall/Outcall | Прозрачность = доверие |
| Submit кнопка | «Request Booking — £350» | Финальное подтверждение суммы перед действием |

**Правила форматирования:** Всегда через запятую (£2,600, не £2 600). Использовать `toLocaleString('en-GB')`.

## 18. РЕКОМЕНДАЦИИ ПО СТИЛЮ И ТИПОГРАФИКЕ

### Типографика:

| Элемент | Шрифт | Размер | Стиль | Цвет |
|---------|-------|--------|-------|------|
| H1 (имена, главные заголовки) | Cormorant Garamond | 48-64px | Regular (НЕ italic) | #FAFAFA |
| H2 (подзаголовки — единственное место для italic) | Cormorant Garamond | 32-40px | Italic | #FAFAFA |
| Body (описания, UI-текст) | Inter / system sans-serif | 16px | Regular | #A0A0A0 |
| Labels, цены, метки | Inter / system sans-serif | 13-14px | Uppercase, tracking 0.05em | #C5A572 (цены), #808080 (метки) |

**Правило:** Italic ТОЛЬКО для H2 и редких акцентных слов. Сейчас italic на всём — убрать.

### Цвет:

```
Золотой (#C5A572):
  ✅ CTA-кнопки (background)
  ✅ Цены (text)
  ✅ Hover-состояния
  ❌ НЕ для заголовков
  ❌ НЕ для секционных меток («Available Now», «Simple Process»)
  ❌ НЕ для декоративных элементов
  ❌ НЕ для sticky bar фона

Остальное:
  Заголовки: #FAFAFA
  Body текст: #A0A0A0
  Вторичный текст: #606060
  Разделители: #1A1A1A
  Фон: #0A0A0A
  Вторичный фон: #111111
```

### Отступы (жёсткая система):

```
Между секциями: 120px desktop / 80px mobile
Padding контейнера: 80px desktop / 24px mobile
Gap карточек: 24px
Карточки: фото p-0 (edge-to-edge), текст p-4
```

### Принцип Hermès: воздух = статус. Чем больше пустого пространства вокруг элемента, тем он ценнее выглядит.

## 20. SEO-АРХИТЕКТУРА ДЛЯ МАСШТАБИРОВАНИЯ

### Текущие проблемы URL:

```
/catalog/[slug]        → «catalog» не несёт SEO-ценности
/escorts-in/[district] → отдельный namespace от /london-escorts
/london-escorts        → основной каталог
/services              → не связан с моделями
```

Два namespace (/london-escorts и /escorts-in/) дробят link equity.

### Рекомендуемая архитектура:

```
/companions/              → Каталог (основной SEO-хаб)
/companions/[slug]        → Профиль модели
/companions/mayfair       → По району
/companions/brazilian     → По национальности (при масштабе)
/services/                → Каталог услуг
/services/[slug]          → Услуга + моделей, кто предлагает
/about                    → Доверие
/contact                  → Контакт + быстрый букинг
/faq                      → FAQ
/blog/                    → SEO-контент (при масштабе)
/join                     → Для моделей
```

### 301-редиректы при миграции:

```
/catalog/:slug        → /companions/:slug      (permanent)
/escorts-in/:district → /companions/:district   (permanent)
/london-escorts       → /companions             (permanent)
```

### Structured Data (JSON-LD) — добавить на каждую страницу:
- LocalBusiness schema на главной
- BreadcrumbList на каждой странице
- FAQ schema на /faq

## 21. ПРИНЦИПЫ HERMÈS, ПРИМЕНИМЫЕ К VIREL

1. **Один продукт — один экран.** Не 6 карточек рядом. Показать 3 — и оставить желание.
2. **Воздух дороже контента.** Убрать 5 секций с главной — станет дороже выглядеть.
3. **Фото продаёт, текст поддерживает.** Перевернуть приоритет: фото = главное, текст = подпись.
4. **Никаких крикливых CTA.** Убрать CAPS LOCK, убрать золотые sticky бары. Premium говорит тихо.
5. **Консистентность = доверие.** Одинаковое качество фото на всех карточках. Никаких селфи рядом со студией.

## 22. ИТОГ

Три вещи, которые дадут 80% эффекта:

1. **Показать цену везде** — на карточках, в hero профиля, в кнопке submit. Это снимает главный барьер и сокращает путь к букингу с 5-7 минут до 2-3.

2. **Убрать мусор с главной** — USP-дубли, «How to Book», SEO-районы, FAQ. 6 секций вместо 11. Каждый экран — одна мысль.

3. **Фото-стандарт** — никаких селфи, барных фото, зеркал с телефоном. Только профессиональные, единый стиль. Это единственное, что отличает premium от classified ads.

Всё остальное — SEO-миграция, rates table, service tags, similar companions — усиливает эффект, но без этих трёх пунктов не работает.

Сайт может быть хорошим. Но сейчас он мешает сам себе.
