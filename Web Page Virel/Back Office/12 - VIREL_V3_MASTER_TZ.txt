## 4. ФАЗА 1 — Крепкий бэк офис

### Блок A — Критические блокеры

#### A1 · Queue Worker → Vercel Cron Jobs
**Приоритет:** 🔴 Блокер · **Время:** 2–3 ч

**Проблема:** Vercel Serverless не поддерживает постоянный фоновый процесс.  
5 задач висят в `queued` бесконечно: `fraud_scan`, `lost_revenue_auto_detection`,  
`operations_feed_auto_close`, `notification_dispatch`, `demand_stats_rebuild`.

**Решение:**
```
1. Создать vercel.json с cron schedule:
   - notification_dispatch  → каждые 5 мин
   - fraud_scan             → каждые 15 мин
   - operations_feed_auto_close → каждый час
   - lost_revenue_auto_detection → каждый час
   - demand_stats_rebuild   → каждые 6 часов

2. Создать API routes для каждого job:
   src/app/api/cron/notification-dispatch/route.ts
   src/app/api/cron/fraud-scan/route.ts
   src/app/api/cron/operations-feed/route.ts
   src/app/api/cron/lost-revenue/route.ts
   src/app/api/cron/demand-stats/route.ts

3. Добавить CRON_SECRET в Vercel env
4. В каждом handler проверять: 
   if (req.headers.get('authorization') !== `Bearer ${process.env.CRON_SECRET}`) 
     return 401
5. Обновлять статус задачи в таблице Jobs после выполнения
```

**Env переменные:**
```
CRON_SECRET=<random 32 chars>
```

**Приоритет:** 🟡 Важно · **Время:** 30 мин

```

    url       = env("DATABASE_URL")
    directUrl = env("DIRECT_URL")   ← добавить
  }

Зачем: предотвращает "too many connections" при масштабировании
```
