# Аудит Back-Office Virel v3.0 vs Museescorts CMS

**Стек:** Next.js 14 (App Router) + PostgreSQL (Neon.tech serverless) + Cloudflare R2 (медиа) + Claude AI API + Vercel. Современный, правильный выбор для быстрого старта.

---

## ЭТАП 1. СТРУКТУРА СИСТЕМЫ

**Полная карта разделов:**

```
CORE          → Action Center, Leads, Inquiries, Clients,
                Booking Requests, Bookings, Models
MASTERS       → Services (79), Call Rates (8), Locations (25 + Transport Hubs)
CONTENT       → Concierge, Blog
OPERATIONS    → Availability, Reviews, Incidents, Fraud Monitor,
                Lost Revenue, Applications, Alternative Offers
BUSINESS      → Pricing, Membership, Payments, Reports, SEO Health, Retention
ANALYTICS     → Dashboard, Owner Analytics, Unit Economics,
                Model Performance, Demand, Staff Performance
SYSTEM        → SLA Monitor, Data Quality, Audit Log, Notifications, Jobs, Settings
```

**27 разделов** против **11 разделов** у Museescorts. Это полноценная операционная платформа, а не просто CMS.

---

## ЭТАП 2. АНАЛИЗ UX

**Сильные стороны:**

Dashboard информативен: разделён на Reception / Revenue / Operations / Models. Показывает Pending Requests с именами и временем, Weekly Revenue с delta vs прошлой неделей, Risk Distribution (13/0/0), AVG Completeness (40% — красный флаг), System Status с реальными статусами (Queue Worker: Not Running).

Availability — лучший модуль в системе. Визуальный недельный/дневной календарь, фильтр по моделям, автодетекция конфликтов с кнопкой "Fix". Это то, чего нет у Museescorts вообще.

Создание модели — 5-шаговый wizard: Identity & Profile → Physical Stats → Rates & Services → Location & Contact → Photos & Publish. Правильная последовательность, не даёт создать «пустой» профиль.

Owner Analytics — полноценная бизнес-аналитика: Total Revenue, Net Margin, ARPU, MRR, Conversion Rate, Lost Revenue, Alt Offer Acceptance, Chargeback Rate, VIP Revenue Share.

Bookings View — карточки с цветными статусами (pending/confirmed/completed), Deposit Required, Total, кнопки смены статуса прямо на странице.

**Проблемы UX:**

Sidebar не имеет группировки по частоте использования. CORE-раздел правильно наверху, но Operations занимает слишком много места. Нет возможности свернуть группы.

**6 страниц возвращают 404** через публичный роутер: `/admin/services`, `/admin/fraud-monitor`, `/admin/seo-health` — это означает, что меню ссылается на `/admin/X`, но роуты зарегистрированы под `/admin/masters/X`. Ошибка в конфигурации навигации.

Clients page и Audit Log — ошибка "Not authenticated" при авторизованной сессии. API-роуты не передают session token корректно.

Model Performance — "Not authenticated" + "No models found" при 10 published models на dashboard. Несогласованность данных между разделами.

AVG Completeness 40% в красном на dashboard — нет прямой ссылки «Fix this». Пользователь видит проблему, но не знает как перейти к её решению.

## ЭТАП 3. ОПЕРАЦИОННЫЕ ПРОЦЕССЫ

**Автоматизировано:** детекция конфликтов доступности, расчёт депозита, статусы бронирований, Risk Distribution моделей, weekly revenue delta, система инцидентов (структура есть), Lost Revenue трекинг (структура есть), AI bio generation (кнопка "Generate with AI" в форме модели).

**Выполняется вручную:** смена статуса бронирования (3 кнопки: DEPOSIT REQUIRED / CONFIRMED / CANCELLED — но нет автопереходов), заполнение SEO-полей (нет шаблонов), публикация модели (последний шаг wizard).

**Где теряются деньги:**

Queue Worker: **Not Running** — это означает все background jobs не выполняются: уведомления клиентам, Telegram-боты (Not Configured), напоминания о бронированиях. При неотправленных подтверждениях конверсия падает напрямую.

Telegram Bot и Telegram Chat — Not Configured. Для ниши, где клиенты используют Telegram как основной канал, это критическая потеря лидов.

Email (Resend) — Not Configured. Нет email-уведомлений ни клиентам, ни операторам.

## ЭТАП 4. ТЕХНИЧЕСКАЯ АРХИТЕКТУРА

**Стек:** Next.js 14 App Router + PostgreSQL Neon (serverless) + Cloudflare R2 + Claude AI + Vercel. Современный и правильный выбор.

**Проблемы:**

Neon serverless PostgreSQL имеет cold start до 500ms при первом запросе после простоя. При активной работе это незаметно, но для редко используемых страниц (Reports, Analytics) создаёт задержки.

Аутентификация нестабильна: 3 разных страницы возвращают "Not authenticated" при авторизованной сессии. Это или session middleware не применяется к API-роутам, или cookie SameSite политика конфликтует с Vercel serverless.

URL-структура непоследовательна: `/admin/bookings`, `/admin/models` — прямые, а `/admin/masters/services`, `/admin/masters/locations` — с префиксом. Навигационное меню ссылается на неверные URL для 6 разделов.

Версия v3.0.0, окружение **Development** в production (показано в Settings). Это означает, что некоторые dev-оптимизации могут быть включены в prod, а NODE_ENV=development отключает ряд production-оптимизаций Next.js.

Slug для Locations: "mayfair", "knightsbridge" — без суффикса -escorts. В отличие от Museescorts (baker-street-escorts), SEO-потенциал локаций в URL не используется.

## ЭТАП 6. HIGH-TECH BEST PRACTICES

**Stripe-паттерны для применения:**

Booking detail page уже близка к Stripe (статус + сумма + deposit + change status buttons). Нужно добавить: Timeline событий (создан / депозит получен / подтверждён / выполнен), как в Stripe Payments.

**Shopify-паттерны:**

Массовые действия (bulk actions) на списках — нет ни в Virel, ни в Museescorts. Activate/Deactivate/Change location для 10+ моделей одним действием.

**Uber-паттерны:**

Availability модуль — лучшее в системе. Развить до диспетчерского вида: какие модели доступны сейчас, map view по локациям Лондона.

**Airbnb-паттерны:**

Completeness score для профиля модели (сейчас есть, но 40% — никаких подсказок что именно заполнить). Нужен breakdown: фото 0/5, services 2/10, bio ✅, location ✅ → Score: 40%.

**Notion-паттерны:**

Inline editing в списках — редактирование статуса/тарифа без перехода на отдельную страницу. Call Rates уже используют inline toggle, но не inline edit price.
