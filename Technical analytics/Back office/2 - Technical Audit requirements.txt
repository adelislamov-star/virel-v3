# BACK OFFICE КАК ОПЕРАЦИОННЫЙ ЦЕНТР БИЗНЕСА + MEMBERSHIP

## Роль

Ты системный архитектор уровня крупного сервиса.
Ты проектируешь back office как инструмент контроля денег, маржи, репутации, дисциплины, антифрода, SEO, SLA и LTV через membership.

Ты не описываешь интерфейсы ради интерфейсов.
Ты выдаёшь проектируемую систему: сущности, поля, статусы, правила, алерты, метрики, API и план релиза.

# II. ONE CLICK ONBOARDING (модуль 5)

Flow: Upload pack → AI Parse → Draft → Review → Publish

Сущности:
OnboardingJob, ModelProfile, MediaAsset, AIExtractionResult

Требования:

* загрузка папки фото
* опциональный файл анкеты
* асинхронная обработка
* confidence score 0..1 по каждому полю + источник (filename, анкета, EXIF, ручной ввод)
* запрещена автопубликация
* дедупликация по хэшу
* media quality scoring (blur, dark, face, composition)
* автокропы в фиксированные размеры
* ретраи и dead-letter для ошибок

State machine:
OnboardingJob: created → queued → processing → needs_review → approved → published → failed
ModelProfile: draft → review → published → hidden → archived
MediaAsset: uploaded → processing → ready → rejected

# XV. MEMBERSHIP & SUBSCRIPTION ENGINE (модуль 21)

## Цель

Увеличить LTV и предсказуемость дохода, разделить клиентов по уровню, встроить привилегии в booking и поддержку, при этом контролировать злоупотребления и маржу.

## Сущности

MembershipPlan

* id
* name
* tier (basic/premium/vip/custom)
* price
* billing_period (monthly/quarterly/yearly)
* currency
* booking_discount_percent
* priority_support_level
* access_rules (например доступ к VIP моделям)
* max_concurrent_bookings
* cancellation_policy_id
* refund_policy_id
* status (active/hidden/archived)

ClientMembership

* id
* client_id
* plan_id
* started_at
* expires_at
* auto_renew
* status (pending/active/past_due/suspended/cancelled/expired)
* payment_provider
* payment_method_id
* next_billing_at
* failed_attempts
* total_paid
* lifetime_value
* abuse_flag

SubscriptionInvoice

* id
* membership_id
* period_start
* period_end
* amount
* currency
* provider
* status (open/paid/failed/refunded/disputed)
* attempt_count

MembershipBenefitUsage

* id
* membership_id
* benefit_type (discount/priority/early_access/vip_access)
* booking_id
* value_amount
* created_at

## State machine Membership

ClientMembership
pending → active → past_due → suspended → cancelled → expired

Правила:

* 3 failed payments → past_due затем suspended
* 2 chargebacks или доказанный abuse → blocked client + membership suspended
* ручная разблокировка только Owner

## Интеграция с Booking Engine

При расчёте цены и правил букинга учитывать membership:

* скидка
* лимит активных бронирований
* доступ к VIP моделям
* priority SLA

Все применения выгод логировать в MembershipBenefitUsage и AuditTrail.

## Антиабьюз

Membership усиливает FraudScore.
Триггеры:

* много отмен + запросы возвратов
* device mismatch
* гео прыжки
* попытки шаринга аккаунта
* постоянные chargeback/dispute

## Метрики Membership (обязательно на Owner Dashboard)

* MRR
* ARR
* Active members
* New members
* Churn rate
* ARPU
* Upgrade / downgrade rate
* Membership to booking conversion
* Revenue uplift
* Discount cost (потеря маржи из-за скидок)
* Abuse rate

Для каждой метрики: формула, источник, период.

## Финансовая устойчивость Membership

* основной платежный провайдер
* резервный провайдер
* smart retry schedule
* grace period
* webhooks обработка событий
* идемпотентность всех платежных операций
